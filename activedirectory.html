This section documents my setup and experimentation with Active Directory in my virtual home lab. I created a Windows Server 2022 Domain Controller (DC), configured users and organizational units (OUs), and connected Windows 10 clients to the domain for testing authentication, GPOs, and red team techniques.

Domain Controller Setup

I began by promoting a fresh Windows Server 2022 instance to a Domain Controller using the Server Manager. The domain was named mydomain.com. I configured DNS alongside AD DS and ensured the server had a static IP address on the internal network.

Organizational Units & User Creation

Using the Active Directory Users and Computers (ADUC) console, I created multiple OUs to simulate departments (e.g., HR, IT, Finance) and added test users and groups within them. I also applied Group Policy Objects (GPOs) to specific OUs to restrict features and enforce policies.

Red Team Testing

With the environment in place, I used tools like BloodHound and PowerView to enumerate AD objects, identify privilege escalation paths, and simulate attacks like credential harvesting and lateral movement in a safe environment.

Skills Learned

  • Windows Server 2022 AD Domain Controller configuration
  • DNS setup and troubleshooting within AD environments
  • Managing users, groups, and GPOs in ADUC
  • Red team enumeration techniques with BloodHound and PowerView
  • Simulated credential dumping and escalation paths

Challenges I Encountered

  • Initial DNS misconfiguration prevented domain joining from clients
  • BloodHound ingestion failed due to WinRM and permissions needing adjustment
  • GPO changes not taking effect until forced via gpupdate /force and system reboots
